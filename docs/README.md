# Docs (Source of Truth)

Everything in this repository is **spec-driven**.

- `/docs/use-cases/` contains one file per use case.
- Each use case file must include a **UC ID** (e.g., `UC-001`).
- If code and docs disagree, this is **drift** and must be fixed before feature work.
- `/docs/architecture/` contains architecture decision records (`ARCH-NNN-*`).
- `/docs/testing/` contains execution guides and test strategy records (`TEST-NNN-*`).
- `/docs/config.md` contains runtime configuration reference for backend/frontend settings.
